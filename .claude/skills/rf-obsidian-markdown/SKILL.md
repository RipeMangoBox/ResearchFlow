# Obsidian Markdown — ResearchFlow Convention

All `.md` files generated by ResearchFlow skills must follow Obsidian Flavored Markdown.
This skill covers only Obsidian-specific extensions; standard Markdown (headings, bold, italic, lists, quotes, code blocks, tables) is assumed knowledge.

## Core Rules

1. **Internal links** — use `[[wikilinks]]` for anything inside the vault.
   Reserve `[text](url)` for external URLs only.
2. **No raw HTML** — Obsidian renders Markdown; HTML tags break reading view and graph indexing.
3. **Frontmatter** — every note starts with valid YAML between `---` fences.
4. **Embeds** — use `![[path]]` for PDFs, images, and note transclusions.
5. **Callouts** — use `> [!type]` syntax for highlighted blocks.

---

## Internal Links (Wikilinks)

```markdown
[[Note Name]]                          Link to note
[[Note Name|Display Text]]             Custom display text
[[Note Name#Heading]]                  Link to heading
[[Note Name#^block-id]]                Link to block
[[#Heading in same note]]              Same-note heading link
```

> When linking to paperAnalysis notes from other notes, use the full relative path:
> `[[paperAnalysis/Category/Venue_Year/Year_Title|Display Title]]`

## Embeds

```markdown
![[image.png]]                         Embed image
![[image.png|300]]                     Embed image with width
![[document.pdf]]                      Embed PDF
![[document.pdf#page=3]]               Embed PDF at page
![[Note Name]]                         Transclude entire note
![[Note Name#Heading]]                 Transclude section
```

ResearchFlow convention for PDF embeds in analysis notes:

```markdown
![[paperPDFs/Category/Venue_Year/Year_Title.pdf]]
```

## Callouts

```markdown
> [!abstract] **Quick Links & TL;DR**
> Summary content here.

> [!note] Title
> Note content.

> [!tip]+ Expanded by default
> Tip content.

> [!warning]- Collapsed by default
> Warning content.
```

### Callout types used in ResearchFlow

| Type | Usage in RF |
|------|-------------|
| `abstract` / `tldr` | Quick Links & TL;DR in analysis notes |
| `note` | General annotations |
| `tip` / `hint` | Aha! Moment highlights (optional) |
| `warning` | Limitations, failure modes |
| `example` | Concrete use cases |
| `quote` / `cite` | Direct quotes from papers |

## Properties (Frontmatter)

Every note must start with YAML frontmatter:

```yaml
---
title: "Paper Title"
venue: CVPR
year: 2025
tags:
  - Category_Name
  - technique-tag
  - status/analyzed
core_operator: One-line description
primary_logic: |
  Multi-line description of the method flow
pdf_ref: paperPDFs/Category/Venue_Year/Year_Title.pdf
category: Category_Name
created: 2026-04-11T12:00
updated: 2026-04-11T12:00
---
```

Rules:
- **Flat only**: all keys are top-level scalars or lists; no nested maps.
- Use `|` for multi-line text values, quoted strings for values containing colons.
- `tags` use YAML list syntax, not inline `#tag` in body (frontmatter tags are indexed by Obsidian).
- `created` / `updated` use ISO 8601 datetime.

---

## Tags

```markdown
tags:
  - Motion_Generation_Text_Speech_Music_Driven
  - diffusion
  - status/analyzed
```

- Tags in frontmatter are preferred over inline `#tag` for searchability.
- Use `/` for nesting: `status/analyzed`, `status/pending`.
- Use `_` for multi-word tags: `Human_Object_Interaction` (not hyphens, to match category names).

## ResearchFlow-Specific Conventions

### Analysis notes (`paperAnalysis/`)
- Frontmatter: full schema (title, venue, year, tags, core_operator, primary_logic, pdf_ref, category)
- Body: Part I / Part II / Part III structure
- Bottom: `![[paperPDFs/...]]` PDF embed
- Related papers linked via `[[paperAnalysis/.../Year_Title|Display]]`

### Collection indexes (`paperCollection/`)
- Frontmatter: minimal (title, updated, type)
- Body: lists of `[[wikilinks]]` to analysis notes, grouped by venue/year/technique
- Purpose: Obsidian graph view navigation for humans

### Wiki layer (`wiki/`)
- `index.md`: structured directory for agent consumption, with `[[wikilinks]]` to analysis notes
- `log.md`: append-only, no wikilinks needed
- `query_pack.md`: query results with `[[wikilinks]]` to cited papers

### Idea notes (`paperIDEAs/`)
- Frontmatter: hypothesis, status, source_papers (list of wikilinks)
- Body: free-form with `[[wikilinks]]` to supporting papers
