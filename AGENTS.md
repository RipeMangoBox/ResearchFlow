# ResearchFlow Agent Guide

ResearchFlow is a **single-core multi-projection** research OS (v3.1).

**Core backend** (PostgreSQL + FastAPI) is the only write target. All other interfaces — Web UI, Claude Code, Codex CLI, Obsidian — are projections.

```text
Paper → L4 analysis → DeltaCard → IdeaDelta → GraphAssertions → Reports/Digests/Directions
```

## Source of truth

- **PostgreSQL (31 tables)**: the canonical data store for papers, analyses, DeltaCards, IdeaDeltas, assertions, evidence, reviews
- **Backend API (67 routes)**: the only write interface
- **MCP server (15 tools, 3 resources, 2 prompts)**: agent integration layer

### Read-only exports

- `paperAnalysis/`: Markdown export of analyses (generated by `compatibility/export_analysis_md.py`)
- `paperCollection/`: index.jsonl + Obsidian navigation pages (generated by `papers-build-collection-index`)
- `paperIDEAs/`: research output notes
- `.claude/skills/`: skill definitions for Claude Code

These directories are **not** authoritative. For programmatic queries, use the backend API.

## Key v3.1 concepts

| Concept | Description |
|---------|-------------|
| DeltaCard | Intermediate truth layer. Captures what a paper changed relative to canonical paradigm. |
| IdeaDelta | Reusable knowledge atom derived from DeltaCard. Must have >= 2 evidence to publish. |
| GraphAssertion | Directed edge with lifecycle (candidate → published). High-value edges need human review. |
| GraphNode | Unified node registry for graph queries. |
| ReviewTask | Audit queue entry. Approval/rejection cascades to target object. |

## Agent integration (MCP)

Agents connect via MCP (`.mcp.json` at repo root). Key tools:

| Tool | Purpose |
|------|---------|
| `search_research_kb` | Hybrid search (keyword + semantic + structured) |
| `search_ideas` | Search DeltaCards/IdeaDeltas by keyword |
| `get_paper_report` | Generate reports (quick/briefing/deep) |
| `get_reading_plan` | Tiered reading recommendations |
| `propose_directions` | Research direction proposals |
| `get_graph_stats` | Knowledge graph statistics |
| `review_queue` | Pending review tasks |
| `submit_review_decision` | Approve/reject reviews |
| `get_digest` | Daily/weekly/monthly research digest |
| `import_research_sources` | Ingest paper URLs |

## Analysis language default

```yaml
analysis_language: zh
```

- `zh`: Chinese analysis prose, English frontmatter keys
- `en`: English analysis prose
- User request overrides this default

## Working rules

1. All writes go to backend API, never directly to Markdown files
2. For queries, prefer `/api/v1/search/*` over reading local files
3. `paperAnalysis/` notes are exports — do not edit them as source data
4. When code and KB work together, keep ResearchFlow as active workspace and link external repos under `linkedCodebases/`
