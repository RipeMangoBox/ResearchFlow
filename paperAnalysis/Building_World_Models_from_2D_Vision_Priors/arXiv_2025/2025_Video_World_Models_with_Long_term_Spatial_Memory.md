# Video World Models with Long-term Spatial Memory

> [!abstract] **Quick Links & TL;DR**
> - **Links**: [arXiv](https://arxiv.org/abs/2506.05284), [Project](https://spmem.github.io/)
> - **Summary**: 这篇工作把视频世界模型的历史从“短窗口像素帧”升级为“全局静态点云空间记忆 + 稀疏历史关键帧情节记忆 + 近期动态工作记忆”，从而显著提升长时回访一致性与相机控制稳定性。
> - **Key Performance**: 回访一致性达到 **PSNR 19.10 / SSIM 0.6471 / LPIPS 0.3069**；用户研究中 **Cam-Acc / Stat-Cons / Dyn-Plaus = 3.626 / 3.385 / 3.401**，均为最佳。

> [!info] **Agent Summary**
> - **task_path**: 源视频/近期帧 + 相机轨迹 + 动作文本 -> 长时、可回访、相机可控的未来视频
> - **bottleneck**: 历史仅以短窗口像素帧存在，无法长期保存静态世界结构，也无法按3D位置检索已见场景
> - **mechanism_delta**: 把“全靠注意力记住过去”改成“静态世界写入全局点云，近期动态留给工作记忆，远期细节由稀疏关键帧跨注意力召回”
> - **evidence_signal**: 反向轨迹回访评测显著提升重建指标，且消融显示三类记忆各自承担不同功能
> - **reusable_ops**: [TSDF静态点云融合, 历史关键帧跨注意力]
> - **failure_modes**: [大位姿跳变会破坏4D重建并导致空间记忆过稀, 长时自回归漂移问题未被解决]
> - **open_questions**: [如何同时兼顾空间一致性与角色一致性, 如何在强动态场景中更稳健地做在线静动分离]

## Part I：问题与挑战

这篇论文要解决的，不是“把视频生成得更长”本身，而是**长时世界一致性**：当镜头在场景里移动、离开、再回到旧位置时，模型能否仍然生成同一个世界，而不是重新编一个相似但不一致的背景。

真正瓶颈在于：现有自回归视频扩散模型通常只看最近几帧。原因不是作者不想看更久，而是 DiT 的注意力代价会随上下文长度快速上升，所以系统只能维持一个很小的滑动窗口。这样做有两个直接后果：

1. **静态环境会被忘记**：建筑、道路、树木、雕像等在回访时容易变形或重绘。  
2. **历史无法按空间检索**：模型只记得“最近发生了什么”，却不知道“当前相机位姿对应世界中的哪一处”。

论文认为，近期一些“更长上下文”方案虽然能压缩或打包历史帧，但本质上仍是**图像级记忆**，不是**世界坐标系中的持久记忆**。所以它们对“回到老地方还能认出同一场景”这件事帮助有限。

**输入/输出接口**：
- **输入**：近期视频帧、目标相机轨迹、动作文本，以及运行过程中维护的长期记忆。
- **输出**：未来视频帧，同时更新静态空间记忆和历史关键帧记忆。

**边界条件**：
- 更适合“**静态背景 + 局部动态主体**”的场景；
- 需要可用的相机/几何重建信号；
- 主要解决**空间一致性与回访记忆**，并不直接解决长时生成的整体 drift。

为什么现在值得做：视频世界模型正从“短视频生成器”走向“交互式世界模拟器”。一旦任务变成可漫游、可回看、可反复访问，短窗口记忆就不够了。

## Part II：方法与洞察

这篇工作的设计哲学很清晰：**不要把所有历史都塞进同一种记忆里，而要按“信息稳定性”拆分存储**。

### 核心直觉

作者的关键变化是把历史信息分成三类：

- **工作记忆（working memory）**：最近几帧，负责短期运动连续性；
- **空间记忆（spatial memory）**：全局静态点云，负责长期静态结构；
- **情节记忆（episodic memory）**：少量历史关键帧，负责远期细节、纹理和身份线索。

这会带来一个本质变化：

> 从“让注意力在像素历史里硬记过去”  
> 变成  
> “让静态世界以几何形式持久存在，动态变化只在短期传播，细节再由稀疏历史帧补回”。

因果上为什么有效：

- **静态结构**本来就应该跨时间稳定存在，所以适合被压缩进可累积的 3D 记忆；
- **动态物体**不稳定，不应该被无脑写入长期记忆，否则会污染世界表征；
- **点云几何**能保位置关系，但细节稀疏，所以再补一个**历史关键帧检索通道**；
- 最终，模型不再需要靠超长注意力去“死记”整个过去，而是把不同时间尺度的信息交给不同记忆载体。

### 机制拆解

#### 1. 空间记忆：把“世界”写成全局静态点云
- 每轮自回归生成后，系统对新帧做在线 4D 重建，得到 point map。
- 用 **TSDF Fusion** 累积这些观测，并利用多帧不一致性自然压制动态物体。
- 留下来的，是一个逐步增长的**全局静态点云**，作为长期空间记忆。

这里的关键不是“多存一点历史”，而是“**只存该长期保存的部分**”。

#### 2. 工作记忆：最近帧继续负责动态连续性
- 生成时保留最近若干帧作为短期上下文；
- 论文实现里使用最后 5 帧来帮助运动平滑、动作延续和短期时序连贯。

也就是说，作者没有试图让点云去负责动态主体；动态还是由最近帧来兜底。

#### 3. 情节记忆：用稀疏关键帧补回细节
- 全局点云能保结构，但往往保不住人物外观、细粒度纹理、历史物体身份。
- 所以系统会监控“当前是否看到了足够多的新区域”；
- 一旦新暴露区域超过阈值，就把该帧存成一个**历史参考帧**。

这些关键帧在生成时通过**历史跨注意力（historical cross attention）**提供远程细节召回。

#### 4. 读取记忆：把几何和历史帧都注入到扩散模型
作者基于 CogVideoX 做改造，读取方式有两条：

- **空间记忆读取**：  
  把全局静态点云沿当前目标轨迹渲染成条件视频，再编码成 latent，通过 **ControlNet 风格的条件分支**注入主干 DiT。
- **情节记忆读取**：  
  历史关键帧也编码成 token，作为 key/value，与当前生成 token 做跨注意力交互。

这等于给模型两种“查过去”的方式：
- 按**相机位姿**查静态世界；
- 按**内容相似性**查历史细节。

#### 5. 数据层支撑：几何增强训练样本
为了让模型学会这种记忆读写，作者从 MiraData 构造了约 **90K** 个结构化样本：
- 97 帧 clip；
- 用 Mega-SaM 恢复相机和深度；
- 用 TSDF 得到静态几何指导；
- 用 Qwen 给未来片段生成动作描述。

所以这篇论文不只是“加一个 memory module”，而是同时补了**训练数据形态**。

### 战略取舍

| 设计选择 | 解决的瓶颈 | 能力收益 | 代价/风险 |
| --- | --- | --- | --- |
| TSDF 过滤后的全局静态点云 | 历史无法长期、按空间索引保存 | 回访一致性更强，相机控制更稳 | 强依赖重建质量；强动态/大视角跳变时会变稀 |
| 最近帧 working memory | 动态主体的短期连续性 | 运动更平滑、动作更接前文 | 只能记短期，解决不了远期遗忘 |
| 稀疏 episodic keyframes | 点云难保纹理与身份 | 细节、角色、历史物体更稳定 | 关键帧选择阈值敏感，覆盖有限 |
| ControlNet式几何条件分支 | 直接重训主干代价高 | 能复用预训练视频扩散骨干 | 条件分支与 backbone 绑定较强 |

## Part III：证据与局限

### 关键证据

**信号 1：反向轨迹回访评测直接验证“记住世界”这件事。**  
作者在 500 条未见过的 MiraData 视频上做 forward/reverse trajectory，让相机位姿被访问两次。这个设置非常对症，因为它直接测试“回到原地时还能否生成同一场景”。结果上，本文方法把回访一致性提升到：
- **PSNR 19.10**
- **SSIM 0.6471**
- **LPIPS 0.3069**

而基线大多停留在 PSNR 约 12、SSIM 约 0.45、LPIPS 约 0.59。这个证据最强，因为它正中论文要解决的核心问题。

**信号 2：用户研究说明收益不只是指标幻觉。**  
20 名有视频/3D/4D 生成经验的参与者，对 14 个代表性案例打分。本文在：
- 相机准确性
- 静态一致性
- 动态合理性

三项 AHR 上都排第一。说明该方法不仅“更像原图”，而且在人感上也更像一个持续存在的世界。

**信号 3：消融证明三类记忆是互补分工，不是冗余堆料。**  
- 去掉 **working memory**，动态主体的平滑运动和短期时序最先受损；
- 去掉 **episodic memory**，远期细节、角色/物体身份保持会变差；
- 三者一起用时最好。

这支持了作者的核心论点：**长期一致性不是靠单一超长上下文解决，而是靠分层记忆分工。**

**平衡看结果**：  
在 VBench 上，本文整体表现最好或接近最好，但并非每个维度都绝对领先。比如 Wan2.1-Inpainting 在成像质量、背景一致性上有些单项更强，说明这篇论文的主要提升点确实是**长期空间记忆与回访一致性**，而不是所有视觉质量维度都全面碾压。

### 局限性

- **Fails when**: 相邻相机位姿跨度过大、轨迹转折过猛、快速穿梭场景（如高楼间快速摆荡）会让 4D 重建出现 ghosting，TSDF 进而误删本应属于静态区域的点，导致空间记忆过稀，最终相机控制和回访一致性下降。
- **Assumes**: 训练和推理都能获得可用的几何估计与相机参数；训练阶段依赖 Mega-SaM、推理阶段依赖 CUT3R；默认场景中存在可长期积累的静态结构；实现依赖 CogVideoX-5B-I2V 预训练主干、90K 几何增强样本、Qwen 动作标注，以及 8×A100 训练资源；当前论文未提供可确认的代码/权重开源。
- **Not designed for**: 直接解决长时自回归 drift、处理大范围拓扑变化或全局动态环境、以及替代专门面向角色一致性的长上下文压缩方法。

### 可复用组件

1. **TSDF 静态过滤融合**：把生成视频中的长期稳定结构抽成全局几何记忆。  
2. **渲染后再条件化的几何分支**：把 3D 记忆变成 2D 条件视频，再接入已有视频扩散模型。  
3. **基于新可见区域的关键帧选择**：用简单启发式构建稀疏 episodic memory。  
4. **历史帧跨注意力**：给“点云保结构但丢细节”的问题补一个远程细节读取通道。
