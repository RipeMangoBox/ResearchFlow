# FashionTex: Controllable Virtual Try-on with Text and Texture

> [!abstract] **Quick Links & TL;DR**
> - **Links**: [arXiv](https://arxiv.org/abs/2305.04451), [Code](https://github.com/picksh/FashionTex), [DOI](https://doi.org/10.1145/3588432.3591568)
> - **Summary**: 这篇工作把全身虚拟试穿从“必须给参考衣服图”改成“文本定款式、纹理块定花纹”，并通过 StyleGAN 潜空间分层编辑与 ID 恢复，在无配对训练数据下生成较稳定的个性化试穿结果。
> - **Key Performance**: 类型编辑 82.75% Accuracy / 69.22 FID；纹理迁移 184.85 FID / 0.3257 LPIPS

> [!info] **Agent Summary**
> - **task_path**: 全身人像 + 服装类型文本 + 上/下装纹理块 -> 保持身份与姿态的试穿图像
> - **bottleneck**: 无配对监督下要同时控制全局服装结构和局部纹理，并避免 GAN 反演误差破坏脸部、肤色与身份
> - **mechanism_delta**: 将 StyleGAN 中层用于文本驱动的类型编辑、细层用于纹理块驱动的纹理编辑，并用相对式 CLIP type loss + 语义感知 ID recovery 收紧非目标区域漂移
> - **evidence_signal**: 类型编辑对比显著优于 StyleCLIP/TediGAN，且移除 type loss 或 text splitting 会造成大幅退化
> - **reusable_ops**: [潜码分层解耦, 相对式CLIP目标构造]
> - **failure_modes**: [反演误差大时仍会出现身份或边界伪影, 罕见服装结构与纹理组合稳定性有限]
> - **open_questions**: [能否去掉每图优化式 ID 恢复, 能否在无解析/无服装类型标签下保持同等可控性]

## Part I：问题与挑战

**What/Why：真正的瓶颈不是“生成衣服”，而是“在真人全身照上，只改该改的服装部分”。**

传统虚拟试穿大多依赖一张参考服装图：用户要先找到一件已经存在、且构图合适的目标衣服，再把它迁移到人物身上。这个交互方式对“自定义设计”并不友好。本文想解决的是更自然的接口：

- **输入**：一张全身人像；
- **控制信号**：
  - 文本：描述上/下装类型，如长袖、短裙、连体衣；
  - 纹理块：给出上/下装局部花纹或材质参考；
- **输出**：同一人物、同一姿态、同一身份下的试穿结果。

### 真正难点

1. **类型与纹理是两种不同层级的编辑**
   - 文本更适合控制全局结构，比如袖长、裙裤、领口；
   - 纹理块更适合控制局部外观，比如条纹、碎花、格纹。
   如果把二者混成一个统一编辑器，常见问题就是：
   - 结构改了但纹理丢了；
   - 纹理贴上去了但衣服形状没变；
   - 脸、皮肤、背景被误改。

2. **没有配对监督**
   现实里很难收集“大规模原图 + 文本指令 + 改后图”的配对数据，所以不能走标准监督式 image editing 路线。模型必须从弱监督或自监督信号里学会“该改哪里”。

3. **真实图编辑依赖 GAN 反演，但反演会伤身份**
   StyleGAN 的潜空间很适合可控编辑，但真实人像先要反演进潜空间。问题在于：
   - 反演和可编辑性之间存在 trade-off；
   - 反演误差会直接体现在脸、肤色、细节上；
   - 虚拟试穿里这类误差非常敏感，因为用户默认“人不能变”。

### 为什么是现在可做

这件事之所以现在变得可行，依赖两个成熟部件：

- **CLIP** 带来了文本-图像对齐能力，能在无配对编辑数据下提供弱语义监督；
- **StyleGAN-Human / 人体生成先验** 提供了相对可解释的人体潜空间，可以把“结构变化”和“纹理变化”尽量分开处理。

### 边界条件

这篇方法默认的场景其实比较明确：

- 输入是**全身、对齐较好的时尚人像**；
- 文本提示默认可拆成**上装 / 下装**两部分；
- 依赖人像解析网络来知道衣服区域在哪里；
- 目标是**2D 视觉编辑**，不是 3D 服装拟合。

## Part II：方法与洞察

**How：作者引入的关键因果旋钮，是“把服装编辑拆到不同潜空间层级和身体部位上”。**

### 方法骨架

整体流程可以概括成四步。

#### 1. 把真人图像投到 StyleGAN 可编辑空间

作者先用反演方法把输入人像映射到 StyleGAN 的 \(W^+\) 潜空间，并把潜码分成三组：

- **coarse**：更偏全局身体/姿态；
- **medium**：更偏服装类型与形状；
- **fine**：更偏纹理、颜色、局部细节。

核心选择是：

- **不动 coarse**
- **用 medium 改服装类型**
- **用 fine 改服装纹理**

这一步本质上是在利用 StyleGAN 的层级可解释性，把“该怎么改”先绑定到不同层。

#### 2. 做一个双分支、part-aware 的服装编辑模块

作者没有让一个网络同时吞掉文本和纹理，而是明确分成：

- **type mapper**：输入文本 embedding，预测对中层潜码的偏移；
- **texture mapper**：输入纹理 patch embedding，预测对细层潜码的偏移。

同时又进一步做了**上装 / 下装拆分**：

- 文本先拆成 upper 和 lower 两部分；
- 每个 mapper 内部都用 part-aware 模块分别处理上装和下装；
- 最后再合并成整体偏移。

这让模型不必用一个混杂表示同时处理“上衣袖长”和“裤子/裙子变化”，减少跨部位串扰。

#### 3. 用“相对式 CLIP 目标”解决无配对类型监督

这是全文最关键的设计之一。

如果直接让编辑结果和目标文本做全图 CLIP 对齐，模型会遇到一个问题：  
CLIP 看到的是整张图，而不是“只该改服装区域”。于是脸、皮肤、背景也可能被牵着走。

作者的做法是构造一个**校准后的目标表示**：

- 从输入图像的 CLIP 图像 embedding 出发；
- 减去原服装类型文本的 embedding；
- 再加上目标服装类型文本的 embedding。

直觉上，这相当于保留“这个人其余部分不变”的信息，只替换服装类型语义。  
所以监督信号从“整图长得像目标文本”变成了“在保持原人物不变的前提下，把原服装改成目标服装”。

#### 4. 用语义感知的 ID recovery 修复反演误差

即便前面编辑对了，真实图反演仍可能让身份变差。作者没有直接把编辑图输出，而是加了一个后处理：

- 先用语义 mask 把编辑后的服装区域贴回原图；
- 再做一次基于生成器先验的细化恢复。

这个模块的作用不是再改服装，而是把“衣服已经变对了，但人脸/肤色/边界不够好”的结果拉回到更自然的图像流形上。

### 核心直觉

1. **从“单一全图编辑”改成“潜层分工 + 身体部位分工”**
   - 改变前：文本和纹理争用同一编辑通道，结构与局部外观容易混淆。
   - 改变后：中层管类型、细层管纹理，上下装再拆开。
   - 能力变化：模型更容易同时做“大结构变化”和“局部花纹控制”。

2. **从“全局文本相似”改成“相对式服装替换目标”**
   - 改变前：CLIP 监督会把整张图都往目标文本推。
   - 改变后：先减掉原服装语义，再加目标服装语义。
   - 能力变化：无配对数据下，监督更聚焦在服装区域，非目标部分更稳。

3. **从“编辑结果直接输出”改成“编辑后再做身份恢复”**
   - 改变前：真实图反演误差直接暴露在最终结果里。
   - 改变后：先语义融合，再利用生成先验恢复。
   - 能力变化：人物身份、肤色和背景更接近输入。

### 为什么这套设计有效

因果上，它其实在同时处理三个不同瓶颈：

- **分布瓶颈**：服装类型和纹理本来就来自不同频率/层级的信息分布；
- **监督瓶颈**：没有配对编辑样本，只能用弱对齐信号，必须把监督缩到“应编辑差异”上；
- **表示瓶颈**：真实图反演不完美，必须允许一个后验修复步骤来补偿生成流形误差。

所以这不是简单“多加几个 loss”，而是明确改变了：
**编辑信号的作用位置、监督信号的定义方式、以及最终结果的生成路径。**

### 战略权衡

| 设计选择 | 带来的能力 | 代价 / 风险 |
|---|---|---|
| 中层改类型、细层改纹理 | 降低结构-纹理串扰，控制更稳定 | 依赖 StyleGAN 层级语义确实可解释 |
| 上装/下装拆分 | 指令更容易落在对应身体区域 | 需要 prompt 可拆分，交互格式受限 |
| 相对式 CLIP type loss | 无配对数据下仍能做类型编辑 | 依赖原服装类型标签与 CLIP 线性可加假设 |
| 语义感知 ID recovery | 缓解真实图反演造成的身份漂移 | 需要解析网络与额外 per-image 优化，推理更重 |
| 冻结 coarse 层 | 更好保留姿态和人体整体结构 | 极大幅度的全局服装几何变化能力受限 |

## Part III：证据与局限

**So what：相对 prior work 的能力跃迁，在于它把“类型编辑”和“纹理编辑”放进同一套可交互试穿流程里。**

### 关键证据信号

#### 1. comparison：类型编辑显著优于通用文本驱动编辑方法
作者把类型编辑与 StyleCLIP、TediGAN 比较。结果显示：

- **FashionTex**：69.22 FID，82.75% Accuracy
- **StyleCLIP**：90.25 FID，22.25% Accuracy
- **TediGAN**：95.44 FID，15.25% Accuracy

**结论**：当服装结构变化较大时，通用文本编辑方法很难在全身时尚人像上稳定改变服装类别；FashionTex 的分层潜码编辑 + 相对式 type loss 更适合这种任务。

#### 2. comparison：纹理迁移优于专门的纹理迁移基线
在纹理迁移上，FashionTex 也优于 TextureGAN、Texture Reformer、DiOr：

- **FashionTex**：184.85 FID，0.3257 LPIPS
- **Texture Reformer**：189.68 FID，0.3687 LPIPS
- 其他方法更差

**结论**：把纹理编辑绑定到细层潜码，再用 patch 级统计约束，会比把整张图当通用 style transfer 更适合服装局部纹理控制。

#### 3. ablation：type loss 和 text splitting 都是关键因果部件
消融非常直接：

- 去掉 **type loss**：
  - Accuracy 从 82.75% 降到 58.75%
  - FID 从 69.22 恶化到 172.01
- 去掉 **text splitting**：
  - Accuracy 进一步掉到 10.75%
  - FID 变为 97.80

**结论**：  
这不是“普通 CLIP 监督 + 一个编辑器”就能做好的任务。  
真正起决定作用的是：

- **监督目标要相对化**
- **控制信号要按上/下装拆分**

#### 4. case-study：ID recovery 能修复真实图编辑后的身份问题
文中展示了去掉 ID recovery 的可视化结果，能看到明显的身份丢失与融合伪影；加上后输出更接近输入人物。

**但要注意**：这一点主要是**定性证据**，缺少单独的强定量身份指标支撑，因此整体证据强度仍应保守评为 moderate。

### 结果该怎么理解

这篇工作的能力提升不是“画质绝对最强”，而是**交互维度更完整**：

- 文本负责“穿什么类型”
- 纹理 patch 负责“穿什么花纹”
- 同一人像上保留身份、姿态与背景

相对已有工作，它最大的价值是把“可设计性”带入了虚拟试穿，而不是只做已有商品图迁移。

### 局限性

- **Fails when**: 输入人像与 StyleGAN-Human 先验偏差较大、对齐或解析失败、或目标服装结构超出训练分布时，编辑结果仍可能出现身份漂移、边界残留、袖子/下摆伪影；少见纹理或极端款式组合稳定性也有限。
- **Assumes**: 依赖预训练 StyleGAN-Human、CLIP、VGG、ArcFace、人像解析网络与反演流程；训练虽不需要配对编辑数据，但仍需要原图的服装类型标签；ID recovery 含每图优化/微调，计算成本明显高于纯前馈方案。
- **Not designed for**: 精确商品级复刻、多视角一致试穿、3D 服装物理贴合、复杂遮挡下的真实试衣模拟，以及完全自由格式、不可拆成上下装的开放式服装设计。

### 证据边界与复现备注

还需要保守看待两点：

1. **只在一个数据集上验证**
   - 按给定校准规则，单数据集结果上限只能给到 moderate。
2. **完整任务没有端到端直接基线**
   - 论文把“类型编辑”和“纹理迁移”拆开分别对比，这是合理的，但也意味着“多模态一体化试穿”本身缺少统一 benchmark。

### 可复用组件

这篇工作最值得复用的，不是具体网络层数，而是以下操作范式：

- **潜码分层解耦**：把结构控制和纹理控制映射到不同编辑层级；
- **part-aware 条件调制**：按上装/下装分别注入控制信号；
- **相对式 CLIP 目标构造**：从“全局语义相似”转为“目标差异替换”；
- **语义融合 + 生成先验恢复**：用于缓解真实图编辑中的反演误差。
